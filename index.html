If I try colspan="16" to fill the whole table's width, then the freeze effect doesn't seem to work at all. This seems to because the cell's width is calculated based on the colspan settings. With colspan="6", the width just happens to fit the table. When colspan="16", the width is too large for the scroll container, and therefor, the freeze property seems to have no effect, because the cell is too wide and you get to scroll to see it. This is one more case, where JS needs to come in and manually mess with the cell's width to make sure things work properly. I.e. if the colspan setting causes the cell to be wider than the scroll container, then the width must be forced to be exactly the width of the scroll container and never larger. Otherwise, the freeze won't work.
